Web Fundamentals Part 1

Introduction to the Web

Understanding how the web works

  • In order to create efficient websites, you must understand the underlying processes of the web.
  • These include: hosting, serving and rendering
  • Internet != (not equal to) the web
  • The web is interactions between clients and servers
  • Servers deliver the content while clients request and show the content
  • Lots web servers to handle the different types of data and services

Working with clients and servers

  • Whenever you visit a website, either through clicking or typing, a request to show the content of that particular website is made.
  • URL - Uniform resource locator
  • DNS - Domain Name Server
  • Request (URL) made to DNS which then is turned into an IP address
  • HTML text is rendered first, followed by CSS/images, and then Javascript
  • Static sites just require the content that is requested.
  • Dynamic sites must be processed by a web app before it can rendered.

The DNS and why it matters

  • A website is assigned a unique IP address
  • Helps identify its location
  • IP addresses are cumbersome to work with, for humans at least.
  • To make it easier, websites are given a unique domain name that is connected to it’s IP.
  • Domain name sent to a name server → server finds address → server returns the IP address → IP address helps find the website you wanted
  • DNS system is a database thats distributed across the world.
  • First DNS database that is contacted is the ISP default DNS.
  • If not found in the default, then it is checked through the root name server
  • DNS caching helps speed up the process.
  • If DNS is not handled correctly, then no one will be able to acces your website.
  • DNS servers refresh their cache every 24 hours.

Internet Protocols

  • HTTP - Hypertext transfer protocol
  • Standard for transferring resources
  • Browser clients and web server are both HTTP(S) based
  • Other common web protocols are: FTP (File transfer Protocol), SMTP (Simple Mail Transfer Protocol), and RTP (Real Time Transport Protocol)
  • HTTP is a one way stateless protocol.
  • FTP differs from HTTP because FTP connects to a file server to help download or upload files. HTTP has trouble transmitting or receiving large files.
  • SMTP is used to send emails while POP (Post Office Protocol) is used for receiving.
  • RTP is for delivering audio and video.
  • For HTTP, once a request is made and received, it is forgotten by both parties.
  • Storing data must be done through another way.
  • Cookies are small text files that the website writes on the client machine.
  • HTTPS is HTTP with a security layer on top.

Anatomy of a URL

  • Web link example: http://www.lynda.com
  • Http: This is the protocol. Tells the browser to use HTTP for the protocol
  • Middle part is the link (www.lynda.com)
  • Also known as resource name.
  • ‘Www’ is the subdomain, while ‘lynda’ is the domain name
  • Domain name is what the browser uses to check with the DNS to see if it exists.
  • ‘Com’ is the top level domain.
  • Subdomain is a mostly a thing of the past. Often times it isn’t required anymore
  • Many different subdomains, not just www. Server admin can create subdomains whenever they want
  • A url also specifies the path on the server and port
  • Most servers default to port :80
  • If needed to include them, just append them after the top level domain with a colon in between
  • Can also pass in parameters to the server using the url
  • Parameters are called name-value pairs.
  • Url parameters are often passed in by form submittals or user clicking on specific areas due to the complexity of them.

How Browsers Work

  • Browsers use the HTTP to communicate with web servers and request content
  • Whenever a link is clicked, the browser sends a request for a document
  • One the browser receives the requested content, it then renders it.
  • Additional content is then rendered if the browser supports the file types ( either by default or through extensions)
  • Client side scripting is then executed when it gets triggered
  • Browser UI - contains visuals such as address bars and interface components
  • Rendering engine parses HTML/CSS and controls the rendering process
  • Built in networking components that are used to communicate with common internet protocols
  • Cookies are often used as the data storage component
  • Javascript interpreter executes JS code
  • Five most common rendering engines: Trident, Gecko, Presto, WebKit and Blink.
  • Blink is based on WebKit but being developed by Google
  • Web designers should test pages in multiple versions of the same browser. This is because older browser versions use older rendering engine versions.

Web Standards and the W3C

  • Back in the day, web browsers built and used proprietary features.
  • This caused headaches and issues since web designers had to make their pages compatible with each browser.
  • Web Standards Project created in 1998
  • Supported the World Wide Web Consortium (W3C)
  • Issued recommendations that everyone should follow
  • Beneficial to learn web standards and stay up to date with them, since it’ll give you a better guarantee that your websites will be consistent across browsers.
  • Best to take into account the web standards when designed, modifying as needed per browser.
  • The W3C website lists the specifications for languages.

Web Server Basics

  • Web server is just a computer with specific software and hardware.
  • All web servers have a IP address. The websites the servers host are all registered through a DNS.
  • Virtual hosting allows a webserver to serve more than just one website.
  • Shared hosting environment - website is hosted on a web server that hosts other websites as well.
  • Dedicated hosting - Your website has it’s own server
  • Websites are stored as a series of files and folders. The HTTP retrieves these files/folders upon request
  • Dynamic websites must first process data from database and run any server side scripts.
  • Web Stack - Stack of software that all works together to serve and process web pages.
  • Not all servers use the same stack.
  • Common web stack includes: OS, web server, database server, and programming language
  • LAMP stack - Linux, Apache, MySQL, PHP/Perl
  • Most commonly used web stack, due to each part of the stack is open-source
  • WAMP stack - Windows, Apache, MySQL, PHP/Perl
  • Make sure that your choice of hosting service has a compatible web stack for your scripting and database needs.

Front-End Technology

Front-End Design

  • Front-end deals with what people see and interact with.
  • Back-end deals with the processes that make things work.
  • Client side and server side often deal with the same things and are used with front and back end
  • Front-end languages include: HTML/CSS and Javascript
  • A lot more development is being done within the browser compared to the past
  • New and powerful Javascript libraries have allowed tasks to be done in-browser instead of on a remote server.
  • Front-end designers are responsible for: visual standards, structuring content, ensuring sites accessibility, typography, page layout, interactivity.

HTML, The language of the web

  • HTML 2.0 specs released in 1995
  • First instance of quote/unquote style
  • HTML 4.0 stabilized the language as it was plagued by lots of proprietary tags during HTML 3.0
  • XHTML 1.0 released in 2000, designed to replace HTML
  • XHTML 2.0 was not backwards compatible
  • WHATWG ( Web Hypertext Application Technology Working Group ) continued development of HTML, eventually creating HTML5
  • HTML5 was recognized by W3C
  • Unlike XHTML 2.0, HTML5 is backwards compatible with previous versions
  • Considered the new standard for HTML

Structuring HTML

  • HTML uses tags to structure content and gives elements meaning
  • At top, you’ll find a document type declaration. It specifies the document type, such as HTML
  • DTD occurs before anything else.
  • Tells the client what type of HTML it’s using so that the client can render it correctly
  • Opening and closing HTML tags - bread of a sandwich
  • Opening/closing head tag - the condiments
  • Head tags include: Meta tags contain keywords or descriptions
  • Links to external resources
  • Head tags don’t contain just one thing. What is found in the head tags is often just to make the page appear as you want it to, or function as you want.
  • The visual elements are actually located in the body tags

Controlling Presentation through CSS

  • CSS - Cascading Style Sheets
  • Controls the presentation of HTML documents
  • Collection of formatting rules, contained in an external file
  • Styles can be found in either external files, embedded in head of HTML docs
  • Styles can overwrite each other if conflicts arise
  • Can deploy CSS by linking to an external CSS file
  • Inside CSS file are a collection of rules. They identify which element on the page is to be styled
  • CSS made of 2 parts - selector and declaration
  • Selector tells which element(s) to style
  • Declaration tells formatting instructions
  • All rules are made of 2 parts - the property and value
  • CSS brings highly modular design
  • Can change HTML and CSS independently of each other
  • Adds portability to content, since CSS can be used to change layout based on screen size/orientation

Client Side Scripting With Javascript

  • Javascript adds interactivity and application functionality to sites
  • Created in 1995 by Netscape
  • Not related to Java in any way
  • Can update data directly, dynamic menus, open new browser windows all in the browser.
  • HTML is static so it can’t do any of that
  • Javascript can be performed on the client side, as well as on the server side
  • For example, Javascript can open up modal window when a user clicks on something.
  • Javascript frameworks are Javascript libraries that contain pre-written functions and objects
  • Help make interactive applications a lot easier for designers
  • Each browser has its own JavaScript interpreter.
  • Important to provide alternative content when JavaScript support is absent.
