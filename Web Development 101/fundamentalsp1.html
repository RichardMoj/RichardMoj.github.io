Introduction to the Web

Understanding how the web works

In order to create efficient websites, you must understand the underlying processes of the web. These include: hosting, serving, and rendering. A common misconception is that the internet is equivalent to the web, when in reality that is not true. The web is interactions between clients and servers, where the servers deliver the content while the clients show the content. There are many different web servers to handle the different types of data and services.

Working with clients and servers

Whenever you visit a website, either through clicking or typing, a request to show the content of that particular website is made.

URL
Uniform resource locator

DNS
Domain Name Server

Request (URL) are made to DNS which then is turned into an IP address. HTML text is rendered first, followed by CSS/images, and then Javascript scripts. Static sites just require the content that is requested. Dynamic sites must be processed by a web app before it can rendered.

The DNS and why it matters

A website is assigned a unique IP address. This helps identify its location. However, IP addresses are cubersome to work with, for humans at least. They are made of a series of numbers, which can be hard to remember. To make it easier, websites are given a unique domain name that is connected to it's IP.

The process goes like this: Domain name sent to a name server -> server finds address -> server returns the IP address

The DNS system is a database that is distributed across the world. First DNS database that is contacted is the ISP default DNS. If not found in the default, then it is checked through the root name servers. DNS cahcing helps speed up the process. If DNS is not handled correctly, then no one will be able to access your website. An important fact to remember is that DNS servers refresh their cache every 24 hours.
