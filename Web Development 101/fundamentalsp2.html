Web Fundamentals Part Two

Front-End Technology

Common Image Types

  • Four most commonly used image formats: JPG, GIF, PNG, SVG
  • Need to find a balance between file sizes and quality of images
  • Higher quality images take more bandwidth and vice-versa
  • Bandwidth serious concern as we have low bandwidth connections but high-resolution displays (Phones/tablets)
  • Browser must support image format to be able to display it. Most do since they have come standards.
  • JPEG - Joint Photographic Experts Group
  • Supports millions of colors - making it ideal for photos with lots of colors
  • However, they are compressed with lossy compression which makes their image quality go down.
  • Compressed each time they are saved.
  • GIF - Graphics Interchange Format
  • Support limited animation. 8-bit which means only 256 colors
  • PNG - Portable Network Graphics
  • Lossless and supports millions of colors
  • High file sizes
  • SVG - Scalable Vector Graphic Files
  • Can resize without loss of image quality
  • Written using SVG markup. Can be easily modified by web designers

What is an API

  • API - Application Programing Interfaces
  • Written to provide a set of instructions and standards for using a specific service
  • Usually written to allow developers an easier time to make an application or use services
  • Allows websites to have functionality far beyond what they normally could do
  • Each API is slightly different
  • APIs limit the type of data they expose
  • APIs can be unreliable depending on who hosts it
  • Creating API connected apps can be time consuming if the API changes while it is still being written.
  • HTML5 introduced APIs such as geolocation, and drag & drop

HTML5 APIs

  • HTML5 was designed to allow integration of many capabilities that web apps needed.
  • Media API allows controlling loading and playback of media
  • Text Track API controls subtitles/captions
  • Drag & drop API allows elements to be draggable within the browser.
  • App Cache API lets persistent web apps to keep working even if they are disconnected
  • Geolocation access location of users
  • Canvas API allows you to draw on the browser.
  • By having a standardized set of APIs, designers can build complex interfaces that function more like an application than a simple website

Web Fonts

  • Previously needed to display fonts that were installed on the client's machine
  • Web-fonts automatically download and temporarily install fonts on the client
  • @font-face technique
  • Requests a font from the web server which is downloaded and temp installed on the browser
  • Had legal issues and concerns with end users
  • Led to development of new font formats such as EOT or WOFF
  • These two compress fonts and optimize for screen usage
  • Can use Typekit to avoid problems with browser implementations or legality
  • Front-end designers can now decide what type of typefaces they want to use, allowing for creation of visually sophisticated web

Back-End Technology

Server-Side Scripting

  • Any scripting or programming that runs on a web server is referred to as server-side scripting
  • Examples: processing form, registering new user, booking a hotel room
  • CGI scripts were used in the earlier days
  • CGI - Common Gateway Interface, created to run scripts from languages such as C+ or Perl
  • Web page first processes the script before page is rendered
  • Scripts are hidden from the user, creating a layer of security between data and source code.
  • Web server most host the application. This can lead to issues when the server is hit by lots of requests, such as slowdown.
  • Client side scripting has become more popular due to the issues that Server side scripting presents

The important of PHP

  • PHP - Hypertext Preprocessor
  • Server side processing language
  • Most widely used language for building dynamic sites
  • Wordpress uses PHP
  • Pairs well with Apache and mySQL databases
  • All 3 are open-source which allowed anyone to create web servers
  • PHP code embedded within a pages HTML
  • Any HTML that ends with .php/.asp/.aspx/.cfm requires it to be processed first
  • Has identifier to distinguish between processing code and HTML code
  • Echo function just displays whatever the result was on the actual page
  • PHP is invisible to the user.

Other Popular server Languages

  • JSP - Java server pages.
  • Server side app tool developed by Sun
  • JSP code embedded directly into HTML
  • Scripts are a combination of XML and java scriplets
  • JSP can be ported to almost any server environment
  • Lots of java platforms and cost of hosting varies greatly
  • .NET describes an entire app framework
  • .NET allows devs to build large scale apps for web/desktop
  • Considerably more complicated than PHP
  • Sites built using .NET requires a web server running windows
  • ColdFusion is added to HTML using own markup language
  • ColdFusion is simpler than other server languages, however it is not free
  • Python not an exclusive web language, but it contains powerful web related libraries
  • Often used to create large dynamic websites and web apps
  • Ruby is a general purpose programming language which can be used for lots of different tasks
  • Choosing a platform is broken down to 3 main things: Cost, hosting availability, ease of use.

Dealing with Data

  • Typically, data can be stored on the client as txt files
  • Browsers are limited to using cookies, small text files.
  • This is due to security concerns
  • Web storage API, introduced by HTML5
  • Websites store key value pairs within the browser.
  • XML is extensible markup language. Can store any data.
  • JSON - Javascript Object Notation
  • Stores simple values in a txt file.
  • Both implement use of JavaScript to store/retrieve data
  • Best way to store data is by using databases
  • Databases contain lots of different tables, which can relate to each other.
  • This allows you to create complex interactions between data
  • Databases aren’t just limited to text either.
  • Indexed databases and Web SQL databases store databases in the browser
  • Relational databases and non-relational databases
  • Relational databases store data in tables that relate to each other based on key columns of information
  • Relational databases driven by something called schema.
  • Schema are troublesome to modify.
  • Non-relational databases do not have schemas and thus do not have this issue

Using SQL Databases

  • SQL - Structured query language
  • The standard language for managing and retrieving information from databases
  • Web apps typically use it to update, create, delete, or retrieve information from relational databases
  • Relational databases most common form of databases used with web apps
  • Store data in one or more tables
  • SQL uses simple logical syntax making it very easy to understand
  • Examples: Insert, delete, update
  • SQL queries are typically found as a string inside web application syntax and passed into the database as the web page is being processed

Content Management Systems

  • CMS - Content management system
  • An app that helps control creation, publishing, and archiving a websites content
  • Allows creation of groups of users who can do different levels of activities on the website
  • Can add site functionality by installing modules
  • Create templates that control layout of page
  • CMS divide the page into sections, which allows multiple headers, footers, or content regions
  • Content is stored in a database with the help of the CMS
  • Use CMS to structure website and build navigation
  • Update content through WYSIWYG editor (what you see is what you get) allows anyone to update and maintain the website’s contents, even non-technical ones
  • Only specific people can edit the content, dictated by the users group level
  • This can be extended to clients as well
  • Most plugins are plug and play that add features to your website
  • WordPress is a popular CMS

Content Delivery Networks

  • Deliver content through a distributed network of servers
  • CDNs distribute over a huge network instead of a single server
  • Often ends up with thousands of servers located all over the world
  • Network shares the load which makes it easier for sites to manage heavy traffic
  • CDNs serve CSS and Javascript libraries primarily.
  • Bootstrap CDN and other several types of CDNs serve specific resources
  • Adding one adds another layer of complexity
  • Using a CDN isn’t preferred when you need complete control over a website

Cloud Services

  • Developers are creating applications and services that fundamentally change the way people interact with the internet
  • Dropbox is a form of Cloud service
  • It allows you to store or retrieve files as long as they are online
  • Can access from anywhere, doesn’t have to be on your local machine
  • Java, Ruby, .NET and Javascript are powering the majority of these services
  • These services require servers which can handle huge amounts of traffic
  • Must also be scalable

What is Github

  • Github is radically changing how people develop, publish and distribute content and applications online
  • It offers a way to store, publish, and version control content
  • Built around Git, an open source version control system created by Linus Torvalds
  • It offers free and private commercial plans
  • Once you’ve uploaded a file, it syncs to your local copy. Whenever a change is made, you must first check the changes.
  • Only the changes are uploaded. At each stage, a version of the previous files is saved.
  • Anyone can access your files and fork them, creating a own personal copy in their github repository. They can then develop separately from you
  • Open source collaborative nature of Github has led to lots of new open source software

Assembly Technologies

JavaScript Libraries

  • HTML, CSS and JavaScript the three core languages for making content on the web
  • Due to the increasing complexity of websites, various assembly technologies have been created to make it easier for web developers to create websites and apps
  • JavaScript library is a collection of pre-written Javascript code.
  • It adds methods and values for commonly performed tasks
  • To use a library, you must include a link to it in the page
  • Often served by CDNs
  • The code that is contained in libraries have been extensively tested by many other users
  • The libraries are also often free
  • Shouldn’t always use them. Can create a lot of overhead if doing simple tasks

Frameworks and boilerplates

  • There are collections of packaged HTML, CSS, and Javascript
  • Main use is to make authoring sites easier
  • Contain CSS files that have predefined rules for layout, typography.
  • HTML contain default starting structure for websites
  • Javascript files provide functionality such as modal windows
  • Boilerplate refers to a set of templates
  • HTML5 is the boilerplate, focusing on providing starter pages
  • Framework are a collection of assets that provide a system for building sites
  • Frameworks add the most overhead, but give the most complete set of tools to build entire front end of projects
  • Focus is creating a stable base, leaving the details such as styling to the designer
  • Useful for creating prototypes, and helps speed up production.
  • Not every company allows the use of them though

CSS preprocessors

  • CSS lacks many features that authors require and have requested
  • CSS preprocessors is a scripting language that allows the extension of CSS with new syntax
  • Compiles the script back into a regular CSS once run through the processor
  • A 2 step process
  • Compile at runtime
  • Benefits include: Adding functionality that is not built in, allow reusable blocks of code
  • Big 3 are: Sass, Stylus, and Less.
  • Reasons not to use: everyone must be familiar with it. Handing it off to others could lead to some hassle
  • Adds another layer of complexity
  • Can lead to code bloating
  • Most of the features are making their way into native CSS
